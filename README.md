 # PriorityQueue

 一个基于 Vue 3 + TypeScript + Vite 的优先级排队工具，用来把一长串“都想做”的事项排成一个明确顺序，并自动拆分成“当前聚焦”和“以后再说”两部分。

 当前版本聚焦 MVP：纯前端、无登录、数据本地持久化、支持多板管理和拖拽排序。

 ## 项目目标

 当待办、目标或方向太多时，真正的问题通常不是缺少事项，而是缺少排序。PriorityQueue 通过“手动排序 + Top N 自动聚焦”的方式，帮助用户减少注意力分散。

 适用场景包括：

 - 个人习惯养成优先级
 - 学习计划重点排序
 - 产品需求排队
 - 团队任务优先级梳理

 ## 当前功能

 ### 事项管理

 - 添加事项，默认插入当前列表底部
 - 编辑单条事项文本
 - 删除单条事项
 - 清空当前板全部事项
 - 一键加载示例数据

 ### 排序与聚焦

 - 支持拖拽排序
 - 支持按钮上移、下移
 - 根据顺序自动计算前 N 项为“当前聚焦”
 - 其余事项自动进入“以后再说”区
 - 支持从“以后再说”移入聚焦区
 - 支持从聚焦区移回“以后再说”
 - 每次排序调整后显示 Toast 轻提示

 ### 配置与复用

 - 支持自定义聚焦数量 N，范围 1 到 10
 - 支持自定义排序依据文案
 - 支持自定义“以后再说”提示文案
 - 支持多排队板创建、切换、重命名、删除
 - 默认自动创建“我的优先级排队”板

 ### 数据能力

 - 使用 LocalStorage 自动持久化所有板数据
 - 支持导出当前板为 JSON
 - 支持从 JSON 导入为新板
 - 支持重置当前板

 ### 体验能力

 - 首次访问自动弹出使用引导
 - 顶部工具栏可随时重新打开引导
 - 支持浅色 / 深色模式切换
 - 支持桌面端与移动端响应式布局
 - 空列表时提供占位提示和示例入口

 当前实现已覆盖 PRD 中 MVP 的核心范围，重点包括：

 - F01-F02：事项增删改

 ## 使用说明

 ### 1. 创建并维护事项

 在主区域顶部输入事项内容，回车或点击按钮即可添加。每条事项支持：

 - 编辑
 - 删除
 - 上移
 - 下移

 ### 2. 调整优先级

 使用拖拽手柄或上下按钮调整顺序。系统会把前 N 项自动视为当前聚焦，其余项自动归入“以后再说”。

 ### 3. 设置聚焦数量 N

 点击顶部工具栏的设置按钮，可以：

 - 修改聚焦数量 N
 - 修改排序依据文案

 修改后会立即重算视图。

 ### 4. 管理“以后再说”

 “以后再说”区域支持：

 - 编辑提示文案
 - 将某项移入聚焦区
 - 在该区域继续拖拽排序

 ### 5. 管理多个排队板

 左侧侧边栏支持：

 - 新建排队板
 - 切换板
 - 重命名板
 - 删除板

 每个板都有自己独立的事项、N 值、排序文案和暂缓文案。

 ### 6. 数据导出与恢复

 顶部工具栏支持：

 - 加载示例数据
 - 重置当前板
 - 导出当前板 JSON
 - 导入 JSON 为新板
 - 清空当前板全部事项

 ## 数据持久化

 项目使用 Pinia + persistedstate 自动将数据写入浏览器 LocalStorage。刷新页面后，以下信息会保留：

 - 排队板列表
 - 当前激活板
 - 每个板的事项顺序
 - N 值
 - 排序依据文案
 - “以后再说”提示文案
 - 深色模式偏好
 - 使用引导已读状态

 ## 部署说明

 当前 [vite.config.ts](vite.config.ts) 中配置了：

 ```ts
 base: '/priority-rank/'
 ```

 这适合部署到仓库子路径，例如 GitHub Pages 的 `/priority-rank/` 路径下。如果部署到站点根路径，需要同步调整该配置。

 ## 目录结构

 ```text
 priority-rank/
 ├─ docs/
 │  └─ PRD-01.md
 ├─ src/
 │  ├─ components/        # 视图组件：侧边栏、输入框、事项卡片、聚焦区、暂缓区、Toast、引导弹窗
 │  ├─ composables/       # 复用逻辑，例如 useToast
 │  ├─ stores/            # Pinia store，集中管理多板与事项逻辑
 │  ├─ types/             # 类型定义与默认文案/示例数据
 │  ├─ App.vue            # 主布局与工具栏入口
 │  └─ main.ts            # 应用入口
 ├─ index.html
 ├─ package.json
 ├─ tailwind.config.js
 ├─ tsconfig.json
 └─ vite.config.ts
 ```
